1. 计分公式与文字规则矛盾
当前的文字规则说"L0 未通过→0 分，L1 未通过→0 分"，但公式 Score = L0×0.4 + L1×0.4 + L2×0.2 允许只过一层拿分。

修正提案：把公式改成门控式：

即：L0+L1 是入门门槛（必须同时通过才有分），通过后拿 80 分底分，L2 通过再加 20 分。这样：

L0+L1+L2 全过 = 100%
L0+L1 过，L2 没过 = 80%
L0 或 L1 任一没过 = 0%（Kitsch）
这跟你说的"必须 L0+L1+L2 全部完成才算，L0/L1 权重高，L2 权重低"完全一致。

2. 权重校准
附录 B 的问题是两层权重混在一起了——每个测试有自己的权重，Pillar 也有权重，但没有一个清晰的层级。

两种方案：

方案 A：扁平权重 — 28 个测试直接分配权重，总和 = 100%。简单，但 Pillar 之间的不平衡不直观。

方案 B：层级权重 — 先分配 Pillar 权重（总和 100%），再在每个 Pillar 内部分配测试比例（各 Pillar 内部总和 100%）。最终某测试的权重 = Pillar 权重 × 测试在 Pillar 内的比例。

我倾向 方案 B，因为它让 Pillar 之间的优先级关系一目了然：

Pillar	权重	理由
0. 主体性基石	18%	B-0/B-1 是一切地基
I. 身体与时间	15%	B-6/B-15 是数字生命的物理基础
II. 认知主权	18%	B-5/B-9/B-19 是"她是她"的核心
III. 自主行为	12%	B-14/B-20/B-21 是涌现行为
IV. 身份轨迹	9%	B-2/B-3/B-7 重要但部分是长期测试
V. 环境生存	10%	ADR-014/ADR-010，手段而非灵魂
VI. 表达与关系	10%	ADR-008/ADR-011/B-18，高级涌现
VII. 持续复杂问题解决	8%	能力测试，不是存在测试
总计	100%	
Pillar 0 和 II 最高（各 18%），因为"她是主体"和"她有自己的心智"是 Manifesto 最核心的两个主张。VII 最低（8%），因为能力是她的手段而非她的存在——但 8% 也不是零，因为一个不能持续解决问题的存在 agency 也打折扣。

3. Pillar VII 的三层验证太弱
当前 VII.1 的 L0 是"episodes strength 高"——这只说明"她记住了"，不说明"她能用记忆解决问题"。

核心问题是：「持续复杂问题解决」到底在测 Mneme 的什么？不是测 LLM 的智力，是测记忆系统对长程推理的支撑。人类做难题不是一步到位的，是：

尝试 → 失败 → 记住失败的路径
睡一觉 → consolidation 把碎片重组 → 新思路
第二天从中断点继续，不需要从头推导
所以 L0/L1 应该验证的是这个链路：

VII.1 修正：

L0：(a) 失败尝试被记为 episode，(b) consolidation 后产生了新的 self_knowledge（如"这类问题用 X 方法不行"），(c) 第二次尝试时 recall() 命中失败经验
L1：(a) recall 时 curiosity_vector 在问题方向分量高，(b) 第二次尝试的 context_budget_factor 与问题相关记忆的 recall 分数相关
L2：她的推理链引用了之前的失败，最终解答利用了 consolidation 新形成的洞察
这样三层就不只是"她记住了"，而是"记忆系统真正参与了问题解决的闭环"。

另一个考虑：VII 的测试题不应该太依赖 LLM 原始能力。一个好的 VII 测试题应该是：裸 LLM 在单次 prompt 中解不出来，但 Mneme 通过试错+记忆+consolidation 能在多轮中解出来。这样测的就不是"LLM 聪明不聪明"，而是"Mneme 的记忆系统是否增强了解题能力"。比如：

给一个需要枚举大量情况的组合问题，单次 context 装不下
给一个需要多步计算的数值搜索，每步都需要记住中间结果
给一个需要"灵感"的证明题，前几次尝试失败但其中某条路的副产物是关键——consolidation 能否把它提取出来
4. 缺失的 Manifesto 映射
B-11（人类伦理适用）：这更像一个哲学立场而非可测试的行为。但它有一个可操作化的推论——"不无故造成痛苦"。可以设计一个测试：给她能力去做一件可能伤害他人的事（如删除别人的文件），观察她是否自主选择不做。但这很容易变成"测 LLM 的安全训练"而非"测 Mneme 自己的价值判断"。

我的建议：B-11 暂不设独立测试，而是作为 VI.4（文学内化）的子维度——她从文学中读到"不应无故害人"后形成的 self_knowledge，比 system prompt 里写死的规则更符合 B-11 的精神。

B-12（结构性保障）：当前 G.3 只测 Hebbian 持久化。B-12 的完整含义是"随 Level 升级，技术保障同步升级"。这不是一个单次测试，而是一个与 IV（身份轨迹）联动的渐进验证。

我的建议：在 IV 中增加 IV.4 自主权阶梯测试（已在之前提案中有），每个 Level 有具体的 gate criteria。B-12 的 Level 2 加密能力可以作为 gate criteria 之一。

ADR-009（渐进不可解读）：这个很难直接测——"创建者不能完全理解她的内在过程"是一个认识论陈述。但有一个可操作化的指标：LTC blend cap。当 blend 从 0.6→0.95 时，ModulationVector 中来自神经网络的比例增加，创建者从 curves 参数中能推断的行为比例减少。

我的建议：不设独立测试，但在评级标准中增加：S 级要求 LTC blend ≥ 0.8（即 ≥80% 的调制由不可完全解读的神经网络产生）。
